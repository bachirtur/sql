📚 Qu'est-ce qu'une Base de Données Relationnelle ?

Une base de données relationnelle est un système qui permet de stocker et d'organiser des informations de manière structurée. Imaginez-la comme un ensemble de tableaux Excel interconnectés !

💡 Définition simple

Une base de données relationnelle organise les données dans des tables (ou relations) qui peuvent être liées entre elles grâce à des clés.

🧩 Les concepts clés

📋

Table (Relation)

Un tableau qui contient des données organisées en lignes et colonnes. Ex: table des livres, table des auteurs.

📝

Attribut (Colonne)

Une caractéristique des données. Ex: titre, auteur, année. Chaque colonne a un nom et un type.

📌

Tuple (Ligne)

Un enregistrement complet. Chaque ligne représente une entrée dans la table. Ex: un livre spécifique.

🔑

Clé Primaire

Un identifiant unique pour chaque ligne. Aucun doublon possible ! Permet d'identifier précisément chaque enregistrement.

🗣️ Le langage SQL

SQL (Structured Query Language) est le langage utilisé pour communiquer avec les bases de données. Il permet de :

🔍

SELECT

Rechercher et afficher des données

➕

INSERT

Ajouter de nouvelles données

✏️

UPDATE

Modifier des données existantes

🗑️

DELETE

Supprimer des données

🔑 Module 2 : La Clé Primaire

La clé primaire est un attribut (ou ensemble d'attributs) qui identifie de manière unique chaque enregistrement d'une table.

📋 Règles de la clé primaire

✓ Chaque valeur doit être unique
✓ Ne peut pas être vide (NULL)
✓ Ne devrait pas changer au cours du temps

❌ Exemple de table avec ERREURS

Observez cette table. Pouvez-vous identifier les problèmes ?

id titre annee
1 Les Misérables 1862
1 Notre-Dame de Paris 1831

🚫 Problèmes détectés :
• L'id 1 apparaît deux fois (Les Misérables et Notre-Dame de Paris)
• L'id 2 apparaît deux fois (Le Petit Prince et Vol de Nuit)

Cela viole le principe d'unicité de la clé primaire !
✅ Table corrigée !
Chaque livre a maintenant un identifiant unique. La contrainte de clé primaire est respectée.

🎯 Pourquoi c'est important ?

🔍 Recherche précise
Permet de trouver exactement UN enregistrement
🔗 Création de liens
Nécessaire pour relier les tables entre elles
🛡️ Intégrité des données
Évite les doublons et les confusions

🔗 Module 3 : Clé Étrangère et Relations

Les tables d'une base de données peuvent être reliées entre elles grâce aux clés étrangères. C'est ce qui rend les bases de données "relationnelles" !

🔑 Clé Étrangère (Foreign Key)

Un attribut dans une table qui fait référence à la clé primaire d'une autre table. Elle crée un lien entre les deux tables.

SELECT * FROM livres JOIN auteurs ON id_auteur = auteurs.id
